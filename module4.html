Module 4 - Defining and Using Playing Cards

Since blackjack is a game with playing cards, you now have to tell the Ruby interpreter what it needs to know about them.  

To do this you first have to define an object called CARD.  This CARD object is a thing that the rest of your blackjack program can use.   An object can contain data and methods.  Data are just a group of variables related to that object and methods are things that the object can do.  

The data for a playing card is:

  • name
    • name of the card as in Ace, Jack, Queen, King, 10, 9, 8, … 1
  • suit
    • either hearts, clubs, spades or diamonds
  • been_played
    • this is an indicator that this card has already been played.   We don't want to be able to deal a player the same Ace of Diamonds twice in one hand

So when we crate a new card it will have a home for these three variables.  These will be the data associated with each card. 

The following Ruby code defines the Card object.  Think of the object like a design of a playing card and an instance as an actual playing card.  The object simply says playing cards have a suit and a value and an indicator as to whether or not the card has been played.  The actual instance is an actual playing card.   The code below in line 1 has the special keyword class which tells ruby we are defining an object named Card.   The special name attr_accessor is short for attribute accessor as attribute is another name for a variable associated with an object.    Line 2 says you can view the values of the data in the variable name, suit, and been_played once you create an instance of this object. 

class Card
    attr_accessor :name, :suit, :been_played    
end

Once you have this code, you can launch the interactive ruby interpreter irb and get a feel for the object.
