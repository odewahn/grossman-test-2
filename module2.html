Module 2: Get Input

At this point, you have just been able to display some output to the screen telling the user that you want to play Blackjack. 

Now, it's time to ask the user for input.

Consider the following Ruby code:

puts "Ready to Deal (y = yes, q = quit)"
input = gets

Line 1 shows the user being prompted with the puts command we learned about in module 1.  Line 2 shows our new thing for today called the gets command.  This is short for get-string. 

Line 2 also has an important Ruby command called the Assignment operator.  The equal sign is used here to indicate to take the thing on the right hand side of the equal sign and move it to a variable on the left hand side of the equal sign.

The general form of the assignment operator is:

variable = value

So we could issue the command:

x = 5

and this would assign the value 5 to the variable x.  Think of a variable as just being a place in the computers memory that has an address just like your mailbox at home.  Inside of that mailbox is a value.   At a given point in time, the mailbox has one value.  Since we can change the value in the mailbox, the mailbox is called a variable whose value can change or vary.  Variables get used all the time in Mathematics; so, if you are familiar with them in mathematics, you will be comfortable with them here.  For a full discussion of variables click here.

Some people see an equal sign and immediately associate it with an equation; in our context, it is not an equation.  The equal sign indicates an assignment of the right hand side of the equal sign to the left hand side.

5 = x

The Ruby interpreter sees the equal sign after the number 5 and realizes that you are trying to move some value in an actual number or literal value.  A literal cannot be changed, it only has the value of 5.   So the Ruby interpreter says it was expecting you to be done when it found the equal sign and you were not done so syntax error is generated. 

When you hit the enter key, a newline character is added to the input. So if you just says

x = gets

and the user enters YES and then hits enter, the value "YES\n" is now stored in x where the \n indicates a new line character.

This \n is usually going to get in your way, so a handy Ruby method called chop is used to chop off the newline character.   If we did not use chop any time we wanted to compare the user response we would have to include the "\n" which is time consuming and clutters up our program.  So we chop off the "\n" and we can focus on the core user response.   Sometimes we do need to care if a newline exists and in those cases we would not chop off the "\n". 

The command x = gets.chop will take the user input string, remove the \n and then store the rest in the variable x.

For our blackjack game we use the following:

puts "Ready to Deal (y = yes, q = quit)"
input = gets.chop

You are now ready to see if the user has entered a y for YES to play Blackjack or a q to indicate they are done with the game and the chop method makes it so you do not have to worry about matching the \n character when determining if the user entered a q or a y.
